Create a simple UI where users will drop in their transactions.csv and
bank_balances.csv. Output: a summary of days of discrepancies, a report of day
to day statement logs of running balance vs bank. Include a download option to
download the report and summary. The downloaded file should be timestamped.
Matching running balance will highlight green. Mismatch will highlight red.
Use flask to make a small backend server and use html to create the front end.

Built a Flask backend + HTML frontend, saved as server.py.

Files created:
  - server.py (Flask server + embedded HTML frontend)

Features implemented:
  - Drag-and-drop (or click-to-browse) file upload for transactions.csv and
    bank_balances.csv
  - Reconcile button (disabled until both files are loaded)
  - Summary banner:
      • Green banner when all balances match
      • Red banner with discrepancy count when mismatches exist
      • Stat boxes: Final Running Balance, Final Bank Balance, Net Discrepancy,
        Days Reviewed, Days with Mismatch
      • List of specific discrepancy dates with amounts
      • Opening balance warning box (yellow) when applicable
  - Day-by-day statement log table:
      • Green row = Match (running balance == bank balance)
      • Red row = Mismatch
      • Grey row = No bank record for that date
      • Columns: Date, Running Balance, Bank Balance, Discrepancy, Status badge
  - Download buttons:
      • "Download CSV" — exports summary + full report as a CSV
      • "Download HTML" — exports a styled, self-contained HTML report
      • Both filenames are timestamped (e.g. reconciliation_20260618_143022.csv)

Run with:
  python server.py
  Open http://localhost:5000

  ── Bugs found and fixed during test authoring ──────────────────────────────

  Bug 1 — main.py (real bug):
    csv.DictReader did not strip whitespace from column headers. A file with
    headers like "date , amount" would produce KeyError: 'date' when the code
    tried to access row["date"]. Fixed by explicitly stripping fieldnames before
    iterating:

      reader = csv.DictReader(io.StringIO(tx_text))
      reader.fieldnames = [f.strip() for f in reader.fieldnames]

  Bug 2 — test helpers (test infrastructure bug):
    The tx() and bk() helper functions naively joined values with commas.
    Amounts like "$1,500.00" contain a comma, so the CSV parser split them into
    extra columns (amount became "$1" instead of "$1,500.00"). Fixed by using
    csv.writer to build the CSV strings, which auto-quotes fields that contain
    commas.
